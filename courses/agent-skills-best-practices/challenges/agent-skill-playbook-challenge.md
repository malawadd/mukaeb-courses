# المطلوب

ابنِ Skill حقيقية لمهمة متكررة في مشروعك، ثم عاملها كما لو أنك ستنشرها لفريق كامل أو تضيفها إلى مستودع Skills مشترك.

## المطلوب داخل التحدي

- اختر workflow متكررًا يستحق Skill مستقلة
- اكتب `SKILL.md` مع `name` و`description` واضحين وقابلين للتفعيل
- أضف أقسام الهدف، والمدخلات، والـ workflow، والحدود، والمخرجات
- أنشئ على الأقل مجلدًا مساعدًا واحدًا مثل `references/` أو `scripts/` إذا كانت المهمة تستدعي ذلك
- اكتب 3 prompts اختبارية واقعية
- راجع المهارة بقائمة تحقق للصيانة والتفعيل
- حدّد إن كانت هذه المهارة ما تزال محلية أم جاهزة للتغليف والمشاركة

## معايير النجاح

- النطاق واضح وغير متضخم
- description يساعد على triggering الصحيح
- توجد خطوات تنفيذية قابلة للملاحظة
- توجد حدود تمنع الاستخدام الخاطئ
- بنية الملفات تخدم المهارة بدل تشتيتها
- توجد خطة تقييم أو prompts اختبارية معقولة

## امتداد اختياري

أضف metadata أو packaging أولي إذا كنت تتوقع مشاركة المهارة مع أكثر من مستخدم، أو صمّم نسخة roadmap توضّح كيف يمكن ضمها لاحقًا داخل Plugin أو marketplace.
