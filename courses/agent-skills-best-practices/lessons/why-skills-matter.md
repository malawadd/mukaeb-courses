# متى تحتاج إلى ملف SKILL.md؟

ليس كل سلوك جديد يحتاج إلى مهارة مستقلة. المهارة تصبح مفيدة عندما تلاحظ أن الوكيل يحتاج إلى **وصفة متكررة** لحل نوع معين من المشاكل، وليس مجرد جملة توجيهية عابرة.

{% callout title="القاعدة الذهبية" tone="tip" %}
إذا كانت المهمة تتكرر، ولها خطوات ثابتة، وتفشل عند التعامل معها بشكل عام أو ارتجالي، فهذه إشارة قوية إلى أنك تحتاج مهارة مستقلة.
{% /callout %}

## متى لا تكفي التعليمات العامة؟

- عندما تحتاج المهمة إلى ترتيب خطوات ثابت
- عندما توجد أخطاء شائعة تريد منعها مسبقًا
- عندما تحتاج إلى أمثلة أو تنسيقات مخرجات محددة
- عندما تريد من الوكيل أن يفرّق بين حالات متشابهة لكن ليست متطابقة

{% tabs %}
{% tab label="تعليمات عامة" %}
مناسبة للقواعد المشتركة بين كل المهام، مثل أسلوب الإجابة أو سياسة عدم لمس ملفات لا تخصك.
{% /tab %}
{% tab label="مهارة مستقلة" %}
مناسبة لتدفق عمل متكرر ومحدد، مثل مراجعة ملفات التخصيص، كتابة `SKILL.md` جديد، أو تنفيذ checklist مخصصة لنطاق معين.
{% /tab %}
{% /tabs %}

## علامات تدل أن الوقت حان لبناء مهارة

{% steps %}
{% step title="تكرار المشكلة" hint="مرتان أو أكثر" %}
إذا شرحت نفس الآلية أكثر من مرة للوكلاء أو للفريق، فالأرجح أنها تستحق مهارة مستقلة.
{% /step %}
{% step title="حاجة إلى شروط تفعيل واضحة" %}
إذا كان الوكيل يجب أن يقرر متى يستدعي هذا السلوك ومتى يتجاهله، فأنت تحتاج قسمًا واضحًا يصف مؤشرات التفعيل.
{% /step %}
{% step title="وجود مخاطر أو قيود" %}
عندما توجد حدود يجب على الوكيل احترامها، مثل عدم استخدام أدوات معينة أو اتباع ترتيب عمل معين، يصبح توثيقها في مهارة أكثر أمانًا.
{% /step %}
{% /steps %}

## مثال ذهني

إذا كنت تريد من الوكيل فقط أن يكون مختصرًا، فهذا لا يحتاج مهارة مستقلة. لكن إذا كنت تريد منه أن:

- يكتشف أن الطلب يتعلق بملفات تخصيص الوكلاء
- يقرأ ملفات معينة أولًا
- يطرح أسئلة محددة عند غياب المتطلبات
- ينتج ملف `SKILL.md` بقالب متكرر

فهنا أنت أمام **سير عمل كامل** وليس مجرد preference.

{% checkpoint title="قبل الانتقال للدرس التالي" %}
- لا تنشئ مهارة لكل شيء.
- أنشئ مهارة عندما تحتاج إلى workflow متكرر ومحدد.
- فرّق بين التعليمات العامة، والتفضيلات، والمهارات المتخصصة.
{% /checkpoint %}
