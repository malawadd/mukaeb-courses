# مراجعة المهارة وصيانتها

إنشاء المهارة ليس النهاية. أي مهارة جديدة يمكن أن تسبب تعارضًا مع مهارات أخرى، أو تنشئ سلوكًا زائدًا عن المطلوب، أو تصبح قديمة بعد تغيّر الأدوات والملفات.

{% callout title="المراجعة ليست تجميلًا" tone="warning" %}
إذا لم تختبر حدود المهارة وحالات عدم استخدامها، فقد تبدو ممتازة على الورق لكنها تسبب تفعيلًا خاطئًا في الاستخدام الفعلي.
{% /callout %}

## Checklist مراجعة قبل الاعتماد

- هل حالة التفعيل محددة وقابلة للملاحظة؟
- هل توجد حالات صريحة لعدم استخدام المهارة؟
- هل workflow يبدأ من جمع السياق الأدنى اللازم؟
- هل توجد خطوة تحقق بعد أول تعديل؟
- هل تتعارض هذه المهارة مع مهارة أخرى موجودة؟
- هل الأمثلة تعكس النتيجة المطلوبة فعلًا؟

## إشارات على أن المهارة تحتاج تحديثًا

{% tabs %}
{% tab label="إشارة من السلوك" %}
الوكلاء بدأوا يفعّلون المهارة في مواقف ليست من اختصاصها، أو يتجاهلونها عندما تكون مطلوبة فعلًا.
{% /tab %}
{% tab label="إشارة من الكود أو الأدوات" %}
تغيّر هيكل المشروع، أو تبدلت الأدوات المطلوبة، أو ظهرت تعليمات أعلى أولوية تجعل بعض خطوات المهارة قديمة.
{% /tab %}
{% /tabs %}

## ما الذي يجب تقليصه لا توسيعه؟

{% accordion title="علامات المهارة المنتفخة" %}
- المهارة تحاول تغطية أكثر من نوع مهمة
- أمثلة كثيرة لكن من دون معيار قرار واضح
- تكرار نفس التعليمات الموجودة أصلًا في ملفات أعلى مستوى
- وصف طويل للهدف مقابل وصف ضعيف للخطوات القابلة للتنفيذ
{% /accordion %}

## دورة صيانة بسيطة

{% steps %}
{% step title="راجع حالات التفعيل الخاطئ" %}
اجمع مثالين أو ثلاثة على لحظات استخدمت فيها المهارة بشكل غير مناسب.
{% /step %}
{% step title="ضيّق trigger أو زد حدود النطاق" %}
صحّح اللغة التي تسمح بالتأويل الزائد بدل إضافة مزيد من الشرح العام.
{% /step %}
{% step title="أعد اختبار workflow" %}
تأكد أن الخطوات لا تزال منطقية مع الأدوات الحالية في المشروع.
{% /step %}
{% step title="حدّث الأمثلة" %}
إذا تغير شكل الملفات أو أسماء الأدوات، حدّث الأمثلة فورًا حتى لا تضلل الوكيل.
{% /step %}
{% /steps %}
