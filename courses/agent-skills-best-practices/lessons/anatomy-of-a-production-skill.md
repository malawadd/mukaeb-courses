# تشريح Skill احترافية

عندما تتصفح مستودعات Skills القوية ستلاحظ بسرعة أن المهارة الناضجة نادرًا ما تكون مجرد `SKILL.md` وحيد. الملف يظل هو القلب، لكن حوله توجد طبقات أخرى تجعل السلوك **أوضح، وأقل غموضًا، وأسهل في الصيانة**.

{% callout title="الدرس الأهم من المستودعات المرجعية" tone="tip" %}
كلما زادت تخصصية المهمة، زادت الحاجة إلى فصل المعرفة طويلة الأمد، والأصول المرجعية، والسكربتات القابلة للتنفيذ عن النص الرئيسي للمهارة.
{% /callout %}

## ما الذي نجده عادة في Skills الجيدة؟

- `SKILL.md` يشرح متى تتفعّل المهارة وما workflow الأساسي
- `references/` تحفظ المعرفة الطويلة أو الجداول أو الوثائق التي لا نريد تحميلها دائمًا
- `scripts/` تنفّذ الخطوات الحتمية والمتكررة بدل مطالبة الوكيل بإعادة اختراعها
- `assets/` تحفظ القوالب أو الخطوط أو الأيقونات أو ملفات الإخراج
- `examples/` أو `rules/` أو `templates/` تقدّم أمثلة domain-specific
- أحيانًا ملفات metadata أو packaging لتوزيع المهارة على أكثر من وكيل أو marketplace

## لماذا لا نضع كل شيء داخل `SKILL.md`؟

{% tabs %}
{% tab label="إذا وضعت كل شيء في ملف واحد" %}
سيصبح الملف طويلًا، صعب القراءة، ومحمّلًا بتفاصيل لا يحتاجها الوكيل إلا في حالات نادرة.
{% /tab %}
{% tab label="إذا استخدمت progressive disclosure" %}
يبقى `SKILL.md` خفيفًا ويرشد الوكيل إلى فتح الملف المناسب فقط عند الحاجة، مثل مرجع لغة معينة أو سكربت تنفيذ محدد.
{% /tab %}
{% /tabs %}

## بنية ذهنية مقترحة

{% steps %}
{% step title="الملف الرئيسي يقرر" hint="Decision layer" %}
دع `SKILL.md` يشرح متى تستخدم المهارة، وما الخطوات الكبرى، وأي ملف يجب فتحه لاحقًا.
{% /step %}
{% step title="المراجع تشرح" %}
ضع التفاصيل الطويلة في `references/` أو `rules/` أو ملفات فرعية، خاصة عندما توجد بدائل متعددة أو جداول أو قوائم طويلة.
{% /step %}
{% step title="السكربتات تنفذ" %}
عندما تتكرر نفس العملية البرمجية أو التحويلية، انقلها إلى `scripts/` بدل وصفها نظريًا فقط.
{% /step %}
{% step title="الأصول تثبّت النتيجة" %}
إذا كان الإخراج يحتاج قالبًا أو خطوطًا أو ملفًا مساعدًا، خزّنه داخل المهارة نفسها.
{% /step %}
{% /steps %}

## مثال عملي على الفرق

مهارة ترجمة README متعددة اللغات لا يكفيها أن تقول: "ترجم الملف". المهارة الأقوى ستتضمن:

- قواعد حفظ الروابط والأكواد كما هي
- مرجعًا لشكل language selector
- naming pattern مثل `README.zh.md`
- checklist للتحقق من anchor links

وهذا هو الفارق بين مهارة "تعرف ماذا تريد" ومهارة "تنجز العمل فعلًا".

{% checkpoint title="الفكرة التي نكمل بها" %}
- `SKILL.md` ليس كل المهارة، بل بوابة المهارة.
- افصل بين القرار، والمعرفة، والتنفيذ، والأصول.
- المهارة الاحترافية تقلل التحميل الذهني على الوكيل بدل أن تزيده.
