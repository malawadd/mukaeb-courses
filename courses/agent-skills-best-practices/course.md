# وصف الدورة

هذه الدورة مخصصة لمن يريد الانتقال من فكرة عامة عن `SKILL.md` إلى **بناء Skill حقيقية قابلة للاستخدام والصيانة والتوزيع** داخل أنظمة الوكلاء الحديثة.

بدل الاكتفاء بملف Markdown قصير، سنعامل الـ Skill كما تظهر في المستودعات القوية: **وصف دقيق للتفعيل، بنية ملفات واضحة، مراجع وسكربتات مساعدة، واختبارات وتقييمات وتغليف للنشر**.

## ستتعلم

- متى تكون `Skill` هي الأداة الصحيحة أصلًا، ومتى تكفي التعليمات العامة أو الـ Plugin
- كيف تصمم الوصف والـ frontmatter بحيث تُكتشف المهارة في الوقت الصحيح
- كيف تحوّل `SKILL.md` من نص إنشائي إلى workflow تشغيلي يمكن تنفيذه والتحقق منه
- كيف تنظّم `references/` و`scripts/` و`assets/` وملفات الميتاداتا بدل حشر كل شيء داخل ملف واحد
- كيف تبني decision trees وحدود نطاق تمنع التفعيل الخاطئ
- كيف تركّب أكثر من Skill داخل نظام واحد من دون تضارب
- كيف تقيّم المهارة باختبارات حقيقية وتراجع التفعيل والأداء مع الوقت
- كيف تجهّز Skill أو مجموعة Skills للتوزيع عبر marketplaces أو مستودعات مشتركة

## مخرجات الدورة

بنهاية هذا المسار سيكون لديك:

- منهج واضح لتحديد ما إذا كانت المشكلة تستحق Skill أصلًا
- قالب عملي لكتابة `SKILL.md` مع frontmatter ووصف triggering قوي
- بنية مجلدات احترافية تشمل `references/` و`scripts/` و`assets/`
- checklist لمراجعة الجودة والصيانة ومنع التضارب
- فهم عملي لكيفية الاختبار والتغليف والنشر
- تحد نهائي يحاكي رحلة بناء Skill من المسودة إلى النسخة الجاهزة للفريق
